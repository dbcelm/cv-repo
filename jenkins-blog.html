Using Jenkins Over A SSL Secured Domain (HTTPS)

Usually Jenkins is ran over port 8080 and eventhough main domain uses SSL, Jenkins domain usually left as it is. This can be a major security overlook as if compromised major secrets like git credentials and application parameters can be left exposed and compromised.

Below is a step-by-step guide on how you can setup your Jenkins over HTTPS:

1. Enable Port 8443 On Jenkins

We will start of by configuring Jenkins to use port 8443 instead of port 8080.

i. Copy the .key and .crt/.pem files for the required domain to any desired directory on server

ii. Next we need to convert these keys to jenkins readable password keystore file

openssl pkcs12 -inkey #filekey.key -in #certificatefile.crt -export -out #keys.pkcs12

iii. On getting password prompt give a strong alphanumeric password having minimum 1 capital letter, 1 special characters and minimum length of 8 characters

iv. Create Java Keystore file at /var/lib/jenkins and empty its default values, ignore if jenkins.jks file already exists at given location

sudo keytool -genkey -alias jenkins -keystore jenkins.jks

sudo keytool -delete -alias jenkins -keystore jenkins.jks

v. On password prompt you can use same password as used in step 2

vi. Navigate back to keys directory and run:

sudo keytool -importkeystore -srckeystore keys.pkcs12 -srcstoretype pkcs12 -destkeystore /var/lib/jenkins/jenkins.jks

vi. Navigate to either /etc/sysconfig/jenkins or /etc/default/jenkins depending on variant and edit jenkins file.

vi. Line to be edited can be found at the very bottom where you have to replace existing content of JENKINS_ARGS with below string:

JENKINS_ARGS="--webroot=/var/cache/$NAME/war --httpPort=-1 --httpsPort=8443 --httpsKeyStore=/var/lib/jenkins/jenkins.jks --httpsKeyStorePassword=#password_you_set"

vii. This will disable jenkins to listen to port 8080 and will use port 8443 by default.

viii. Run service jenkins restart and test URL over https:// and port 8443. If working we can move to configure reverse proxy.

2. Configuring Reverse Proxy

Now we will configure reverse proxy service, I'm taking Nginx here as example to route traffic to desired domain and port.

i. Install nginx web server with:

sudo apt update

sudo apt install nginx

ii. Navigate to /etc/nginx/sites-enabled/ and edit default file by adding two new server blocks one for port 8080 > 8443 redirect and other for port 80 > 8443 redirect:

server {

listen 8080;

listen [::]:8080;

server_name jenkins.#example.com;

return 301 https://jenkins.#example.com:8443$request_uri;

}

server {

listen 80;

listen [::]:80;

server_name jenkins.#example.com;

return 301 https://jenkins.#example.com:8443$request_uri;

}

iii. Restart nginx and we are good to go

sudo systemctl restart nginx
